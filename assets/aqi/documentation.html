AIR QUALITY SEARCH

Documentation

Step 1: API Calls and Controls

The first thing I did was use the AirVisual API to create a system where users selected first a Country then State within that country and finally a City within that State.

Step 2: API Display and Selection

I next got the data at that specific location and displayed it on the page. I also added some features that let users so back to select a new State or City within the already specified parameters so that users didn't have to go back to the country search every time.

Step 3: Another API!

Using the LocationIQ API I displayed a map at the latitude and logitude of the selected city alongside the data to give a bit more context to the data.

Step 4: Storage and Defaults

I then created a system that could store the users selected city, state, and country when the webpage was refreshed. As a little bonus, the default loaded city when you open the page for the first time is the closest city to your iP adress

Step 5: Searching and Error Checking

The final thing I added to the app was the search box capabilities that let users seacrh for a specific Location in the list. If no city is found or the city selected has no data a small error message indicting such will appear.

Step 6: Style

To style the website I used the skeleton framework with some additional css applied to make it feel more like a map style as well as a lighter font face. The pages a displays organize and display appropriatly at any screen size
